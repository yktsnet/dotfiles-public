# readme-i18n

既存の日本語 README.md を原本として英語版を生成/更新する。

## 対象の決定

1. `$ARGUMENTS` にパスがあればそのディレクトリの README を対象にする
2. カレントディレクトリに `README.md` があればそれを対象にする
3. どちらもなければ、1階層下のサブディレクトリを走査し `README.md` を持つディレクトリを列挙する。対象一覧をユーザーに提示し、確認を得てから一括処理を開始する。

一括処理の場合、各ディレクトリに対して以下の手順を繰り返す。

## 手順（ディレクトリごと）

### 1. README.md を読む

対象ディレクトリの `README.md` のみを読み込む。CLAUDE.md・context/・skills/ など他のファイルは読まない。翻訳に必要な情報は README.md の中に全てある。

### 2. 言語リンクの挿入（未挿入の場合のみ）

README.md の先頭（H1 の直前）に以下を挿入する。既に同等のリンク行があればスキップ。

```
[🇯🇵 日本語](README.md) | [🇬🇧 English](README.en.md)
```

### 3. 英語版の生成/更新

- `README.en.md` が存在しない場合: README.md の内容を自然な英語に翻訳して新規作成
- `README.en.md` が既に存在する場合: README.md の差分を反映して更新（英語版独自の加筆があれば保持）
- 英語版の先頭にも同じ言語リンクを挿入（リンク先は逆）:

```
[🇯🇵 日本語](README.md) | [🇬🇧 English](README.en.md)
```

### 4. 翻訳ルール

- 技術用語・コマンド・コードブロック・パス・URL はそのまま残す
- Mermaid 図のノードラベルは英訳する
- テーブル構造・バッジ・リンクは維持
- 見出しレベル・構成順序は原本と一致させる

### 5. 出して止まる

変更をワーキングツリーに残し、追加/更新した内容を要約して報告する。
一括処理の場合は処理したディレクトリ一覧と各ステータス（新規作成/更新/スキップ）を表示する。
コミット/push はユーザーの指示があったときのみ。
