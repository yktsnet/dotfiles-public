# jp-writing-code

`.astro`、`.html`、`.vue`、`.tsx` 等のソースファイルに含まれる日本語テキストを修正する。

規範は同一プラグイン内の ../jp-writing/SKILL.md の「規範」セクションに従う。毎回読みに行くこと。

## 1. ファイルを選ぶ

ユーザーにどのファイルを対象にするか質問する。

ユーザーがファイルを指定しない場合は、カレントディレクトリから日本語テキストを含みそうなファイルを探索して候補を出す:

```bash
grep -rl '[ぁ-ん]' --include='*.astro' --include='*.html' --include='*.vue' --include='*.tsx' --include='*.jsx' --include='*.svelte' .
```

一覧してユーザーに選ばせる。

## 2. 修正する

1. 対象ファイルを読む
2. `../jp-writing/SKILL.md` の規範セクションを読む
3. ファイル内の日本語テキスト部分のみを規範に照らして修正する
4. コード・属性・変数名・コメントのロジック部分には手を付けない

### やらないこと

- HTML/JSX の構造、属性、コンポーネントの変更
- コードロジックの変更
- 著者の主張・論旨の変更。変えるのは表現だけ
- 判断に迷う箇所は修正せず、報告に回す

## 3. 報告する

修正後に以下を報告する:

- 修正した箇所と理由（規範のどのルールか）
- 迷って手を付けなかった箇所（あれば）

## 4. コミット・push はしない

変更はワーキングツリーに残す。コミット/push はユーザーの指示があったときのみ。
